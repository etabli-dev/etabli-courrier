# Connecting Microsoft 365

`courrier` connects to Microsoft 365 mailboxes via **OAuth 2.0 + IMAP/SMTP
XOAUTH2** — Microsoft retired Basic Auth for legacy protocols on Oct 1 2022
and disabled SMTP AUTH per-mailbox by default afterwards, so this is the only
path that survives. M8 ships the full vertical.

## What's in the binary

- **`core/auth/`** — vendor-neutral OAuth surface. `OAuthProvider` interface,
  `AppAuthOAuthProvider` (production wrapper around `flutter_appauth`,
  authorization-code + PKCE, **no client secret**), `OAuthTokenStore`
  (refresh token in `flutter_secure_storage`, access token in memory only).
- **`modules/mail/providers/microsoft365/`** — the optional provider module.
  Every Microsoft-specific identifier lives here. The
  `isolation_test.dart` test fails the build if a leak shows up elsewhere.

## Provider model

1. **`M365Config`** holds the PREFLIGHT-locked endpoints:
   - IMAP `outlook.office365.com:993`
   - SMTP `smtp.office365.com:587`
   - Authority `login.microsoftonline.com/<tenant>` (default `common`)
   - Scopes `offline_access`, `https://outlook.office.com/IMAP.AccessAsUser.All`,
     `https://outlook.office.com/SMTP.Send`
   - Redirect URI `dev.etabli.courrier://oauth` (matches the iOS
     `CFBundleURLTypes` and Android `appAuthRedirectScheme` from M0).
2. **`M365Backend`** wraps the M6 `ImapMailBackend` with an OAuth refresh-
   on-`UnauthorizedError` loop. Every call routes through `_withRefreshRetry`
   so a token mid-call expiry is invisible to the caller.
3. **`TenantStateClassifier`** maps OAuth + IMAP/SMTP errors onto one of the
   PREFLIGHT-listed friction cases — never a silent crash.

## Tenant friction is first-class UX

The audit gate says: *"tenant-consent error states render"*. We emit five
explicit `TenantState` types:

| State                          | Trigger                                                     | What the UI shows                                                |
| ------------------------------ | ----------------------------------------------------------- | ---------------------------------------------------------------- |
| `AdminConsentRequired`         | OAuth `consent_required` (AADSTS65001)                      | "Your IT admin needs to grant access" + admin-docs CTA           |
| `AppBlockedByTenantPolicy`     | OAuth `access_denied` / `unauthorized_client`               | "Your tenant blocks third-party mail apps" + admin-docs CTA      |
| `ImapDisabledByAdmin`          | IMAP `NO LOGINDISABLED` / "IMAP is disabled"                | "IMAP is turned off on this tenant" + admin-docs CTA             |
| `SmtpAuthDisabledForMailbox`   | SMTP `535 5.7.139` / "SMTP AUTH has been disabled"          | "Modern tenants disable SMTP AUTH per-mailbox by default" + CTA  |
| `TenantAuthFailedGeneric`      | Anything else                                                | "Connection failed" + Try again                                  |

The Connect screen surfaces the right card automatically — see
`modules/mail/providers/microsoft365/ui/connect_microsoft_screen.dart`.

## F-Droid hygiene

The M365 module triggers the **F-Droid `NonFreeNet` anti-feature** (allowed,
informational). Modularity contains it: the rest of the binary still ships
F-Droid-clean. The enforcement points:

- The `isolation_test.dart` test scans every file under `lib/` and **fails the
  build** if `login.microsoftonline.com`, `outlook.office365.com`,
  `outlook.office.com`, or `smtp.office365.com` appear outside
  `lib/modules/mail/providers/microsoft365/`.
- `.github/workflows/license-gate.yml` runs an advisory grep on the same
  identifiers; M11 promotes that grep into a hard CI gate.
- The banned-package check refuses to add `msal_auth` (Microsoft's
  proprietary native MSAL wrapper). `flutter_appauth` wraps the open-source
  AppAuth SDKs (Apache-2.0 upstream) and stays GREEN.

## Setting up your Entra app (one-time, project owner)

Per PREFLIGHT.md §2: register ONE Microsoft Entra ID application:

1. Azure portal → App registrations → **New registration**.
2. **Supported account types** — *Accounts in any organizational directory*
   (or *all tenants* if you also want personal MSAs).
3. **Redirect URI** — Public client / native; URI `dev.etabli.courrier://oauth`.
   Add the same URI for iOS and Android (Entra accepts the scheme once).
4. **Authentication** → **Allow public client flows** = Yes.
5. **API permissions** → Microsoft Graph and Office 365 Exchange Online:
   - `offline_access` (Delegated)
   - `IMAP.AccessAsUser.All` (Delegated)
   - `SMTP.Send` (Delegated)
6. Pursue **publisher verification** before public release — removes the
   *"unverified app"* prompt in managed tenants.
7. Copy the application (client) ID into `secrets.json` as `M365_CLIENT_ID`.
   The ID is public; this is the same posture Thunderbird and K-9 use.

End users register nothing.

## Live gate (opt-in)

The M8 gate is *a live M365 mailbox connects via XOAUTH2, sends + receives,
token refreshes after expiry*. The unit + widget suite proves the
plumbing — the live test is opt-in once `secrets.json` carries
`M365_CLIENT_ID` (public is fine), a sandbox dev-tenant mailbox, and you've
enabled IMAP + SMTP AUTH on that mailbox in Exchange admin.

## Beyond v0.1.0

- **Microsoft Graph backend (v2)** — unify mail + calendar + contacts behind
  the existing interfaces via Graph REST. Required before EWS retires from
  Exchange Online (~Oct 1 2026 — verify against the Exchange Team blog).
- **Gmail seam (deferred to v2)** — Google's restricted-scope
  `mail.google.com` requires a CASA security assessment we're not taking
  on for v0.1.0. The `OAuthProvider` interface is ready; only a
  `GmailConfig` analogue + a `GmailBackend` analogue need to land.
